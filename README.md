# Minecraft服务器监控插件

这是一个基于 AstrBot 框架开发的 Minecraft 服务器监控插件，支持为多个 QQ 群独立配置服务器，并定时检测状态变化后自动发送通知。\
本插件基于原作者cryfly666为灵感，添加了多项实用功能。

## 功能特性

- 🔄 **多群独立监控**：每个 QQ 群可独立配置服务器，互不影响
- 📊 **Java / Bedrock 双版本支持**：通过 `server_type` 指定服务器类型
- ⚡ **智能变化检测**：只在玩家加入/离开或服务器上/下线时发送通知
- 📝 **独立日志**：WebUI显示本插件的运行日志，方便查询
- 💬 **一言集成**：查询与通知消息附带一言句子

## 首次使用
1. 安装插件后，需要先在插件配置中开启全局监控。
2. 在 AstrBot WebUI「插件配置」中添加群配置。
3. 保存配置后，插件会自动加载所有已保存的群配置。

## 配置说明

插件配置分为**全局配置**和**每群配置**两部分。

### 全局配置

在 AstrBot WebUI「插件配置」中设置：

| 配置项 | 说明 | 类型 | 默认值 |
| --- | --- | --- | --- |
| `check_interval` | 监控检查间隔（秒） | 整数 | 10 |
| `enable_auto_monitor` | 插件加载后是否自动启动监控 | 布尔值 | false |

### 每群配置

在 AstrBot WebUI「插件页」→「Minecraft服务器监控」中管理，每个群包含以下字段：

| 配置项 | 说明 | 类型 | 默认值 | 必填 |
| --- | --- | --- | --- | --- |
| `group_id` | QQ 群号 | 字符串/数字 | - | ✅ |
| `name` | 显示用服务器名称 | 字符串 | "Minecraft服务器" | ❌ |
| `server_ip` | 服务器 IP 或域名 | 字符串 | - | ✅ |
| `server_port` | 服务器端口 | 整数 | 19132 | ✅ |
| `server_type` | 服务器类型：`java` 或 `bedrock` | 字符串 | bedrock | ✅ |
| `enabled` | 是否启用该群监控 | 布尔值 | true | ❌ |

## 可用指令

| 指令 | 说明 | 使用场景 |
| --- | --- | --- |
| `/start_server_monitor` | 启动全局定时监控任务 | 所有已启用且配置完整的群都会被轮询 |
| `/stop_server_monitor` | 停止全局定时监控任务 | - |
| `/查询` | 立即查询当前群配置的服务器状态 | 仅在群聊中可用 |
| `/重置监控` | 重置当前群的状态缓存 | 下次检测视为首次检测 |

## 监控逻辑

1. 插件启动后会加载所有已保存的群配置。
2. 调用 `/start_server_monitor` 或在全局配置中开启 `enable_auto_monitor` 后，监控任务开始运行。
3. 每个检查周期内，插件会并发查询所有已启用群对应的服务器状态。
4. 使用每群独立的状态缓存（`ChangeTracker`）对比前后两次数据，仅在以下情况发送通知：
   - 首次检测到服务器状态
   - 服务器上线或离线
   - 有玩家加入或离开

## 通知消息格式

检测到变化时，机器人会向对应群发送：

```
🔔 服务器状态变化：
[变化描述]

📊 当前状态：
🟢 服务器: [名称]
📝 MOTD: [服务器MOTD]
🎮 版本: [版本]
👥 在线玩家: [当前]/[最大]
📋 玩家列表: [玩家1, 玩家2, ...]
🔧 类型: [Java版/基岩版]
🕒 更新时间: [时间]

💬 [一言句子]
```

## WebUI 功能

插件在 AstrBot WebUI 中提供以下接口，便于管理和排障：

- 查看/新增/更新/删除群配置
- 查看监控运行状态（是否运行、已启用群数、最近检查时间等）
- 查看和清空运行日志

## 数据来源

服务器状态通过 [mcstatus.io](https://api.mcstatus.io/v2/status) 公共 API 查询，插件本身不直接连接游戏服务器。

## 注意事项

- ⚠️ 插件需要能访问外网以调用 mcstatus.io API 和一言 API
- ⚠️ 当前版本主要适配 `aiocqhttp` 平台，请确保机器人已在目标 QQ 群中
- ⚠️ 建议将 `check_interval` 设置为 10~60 秒，避免过于频繁的请求
- ⚠️ 只有在 WebUI 中为当前群添加了配置，`/查询` 和监控通知才能正常工作
- ⚠️ `/start_server_monitor` 启动的是全局任务，单个群可通过 `enabled` 开关控制是否参与监控

## 常见问题

**Q: 为什么自动监控没有启动？**\
A: 检查全局配置中 `enable_auto_monitor` 是否为 `true`，以及是否至少有一个已启用且填写了 `group_id`、`server_ip`、`server_port` 的群配置。

**Q: `/查询` 提示当前群未配置监控？**\
A: 需要先在 WebUI「Minecraft服务器监控」页面添加该群的配置。

**Q: 如何清除某个群的监控状态？**\
A: 在该群中发送 `/重置监控`，该群的状态缓存会被重置，下次检测视为首次检测。

**Q: 消息没有发送到群里？**\
A: 检查群号是否正确、该群配置是否启用、机器人是否有发送权限，以及该群是否触发过任意一条消息以让插件缓存会话信息。
